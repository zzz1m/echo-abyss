## 1. 游戏概述

### 1.1 基本信息

| 项目 | 内容 |
|------|------|
| **游戏名称** | 回响深渊 (ECHO ABYSS) |
| **游戏类型** | 俯视角 Roguelike 动作游戏 |
| **游戏平台** | PC (Windows/Mac/Linux) + HTML5 Web |
| **目标用户** | 15-35岁，喜欢暗黑风格和策略动作游戏的玩家 |
| **游戏时长** | 单局30-60分钟，完整通关2-4小时 |
| **引擎选择** | Godot 4.x |

### 1.2 核心体验

```
【游戏核心循环】

    探索 → 遭遇 → 战斗 → 收集 → 成长 → 死亡/前进
      ↑                                          ↓
      └──────────────────────────────────────────┘
```

### 1.3 游戏特色

| 特色 | 描述 |
|------|------|
| **声音可视化** | 所有声音都有对应的视觉表现，声波、频率都是可见的存在 |
| **Roguelike结构** | 永久死亡，但保留回响币和解锁内容 |
| **三职业系统** | 聆听者、喧嚣者、沉默者，三种完全不同玩法 |
| **氛围沉浸** | 每个区域有独特的声音主题和环境氛围 |
| **分支结局** | 两个结局，玩家的选择影响世界命运 |

## 2. 世界观与故事

### 2.1 核心概念

#### 回响 (The Echo)

> 回响是所有生命消逝时留下的声音痕迹。

- 每一次心跳、每一句言语、每一次呼吸
- 当生命终结，声音并不会消失
- 它们汇聚、凝结、最终形成一种神秘的能量

#### 回响深渊 (The Echo Abyss)

> 亿万年来的声音汇聚成一个巨大的深渊。

- 深渊的边缘不断"外溢"
- 被回响碎片沾染的生物会变得狂暴
- 普通人无法听到深渊中的声音，但聆听者可以

### 2.2 历史时间线

```
【远古时代】- 回响开始形成
    ↓
【第一个千年】- 深渊诞生
    ↓
【第一个万年】- 深渊开始外溢
    ↓
【五百年前】- 聆听者协会成立
    ↓
【一百年前】- 协会全灭，只剩玩家
    ↓
【现在】- 玩家开始潜入深渊
```

### 2.3 聆听者协会

```
【协会宗旨】
1. 潜入深渊，收集回响碎片
2. 阻止深渊继续扩张
3. 寻找封印或消灭深渊的方法

【组织结构】
- 大聆听者（1人）- 已牺牲
- 聆听者（5人）- 全部死亡
- 见习聆听者（若干）- 仅玩家存活

【玩家角色】
身份：最后的见习聆听者
年龄：20岁左右
背景：被大聆听者收养，对声音有特殊感知能力
```

### 2.4 核心人物

#### 玩家角色

| 属性 | 描述 |
|------|------|
| **名字** | 玩家自定义（默认：艾文/Evan） |
| **身份** | 最后的聆听者 |
| **特征** | 眼睛被眼罩遮住（能看到声音） |
| **动机** | 完成协会遗志，揭开回响真相 |

#### 关键NPC（深渊中的声音残响）

| NPC | 描述 | 作用 |
|------|------|------|
| **大聆听者的残响** | 协会创始人，声音的回响 | 给予任务指引，讲述背景 |
| **第一个受害者** | 深渊中最古老的声音 | 揭示深渊的真正起源 |
| **深渊之心** | 由无数声音凝结而成 | 最终BOSS/最终对话对象 |

### 2.5 故事线

#### 序章：协会遗迹

```
场景：协会总部废墟
内容：
- 玩家回到协会遗迹
- 发现大聆听者留下的最后记录
- 获得"聆听者之眼"（核心道具）
- 了解深渊外溢的危机
- 进入深渊

关键对话：
"你来得正好，深渊的扩张比我们预想的更快。
我不是协会的第一个牺牲者，也不会是最后一个。
但你不一样，艾文……你的眼睛能看见我看不见的东西。
现在，去完成我们未竟的使命。
去深渊，找到答案。"

— 大聆听者 艾琳娜·声渊
```

#### 第一幕：回响森林

```
区域：森林表层
主题音：扭曲的鸟鸣、风声
关键发现：
- 早期动物被回响侵蚀
- 遇到其他协会成员的残响
- 首次遭遇回响守卒

叙事碎片：
"这些树的年轮里，记录着无数的声音。
有欢笑，有哭泣，有永恒的沉默。
它们曾是森林的守护者，现在……
现在它们只会回应深渊的召唤。"
```

#### 第二幕：熔岩回响

```
区域：地下熔岩层
主题音：岩浆冒泡、火焰噼啪
关键发现：
- 温度即是声音的频率
- 发现协会成员"喧嚣者托尔"的遗物
- 遭遇颤音魔女

叙事碎片：
"声音在高温中会变得尖锐。
愤怒的声音燃烧得最旺，
而恐惧的声音……早已汽化消失。
托尔就是在这里消失的。
但他的战鼓声，至今仍在回响。"
```

#### 第三幕：深海低语

```
区域：地下海洋层
主题音：水压、泡泡、深海怪声
关键发现：
- 水压会放大声音
- 发现协会成员"沉默者涟"的遗物
- 了解深渊与海洋的关联

叙事碎片：
"在深海，声音传播得比光更远。
涟曾说她能听到深渊最深处的声音。
不是嘶吼，不是尖叫，
而是……低语。
她在消失前告诉我：'深渊会说话。'"
```

#### 第四幕：骸骨圣殿

```
区域：骸骨与废墟层
主题音：骨头碰撞、幽冥低语
关键发现：
- 这里是"永恒的声音"所在
- 所有死者的声音都会留在这里
- 面对内心恐惧的试炼

叙事碎片：
"每一个声音都有归宿。
森林的归宿是风，海洋的归宿是浪，
而这里的归宿……是永恒。
欢迎来到声音的墓地，艾文。
这里埋葬着无数被遗忘的呐喊。"
```

#### 终章：深渊之心

```
区域：深渊核心
主题音：所有声音的混合、沉默与轰鸣交替
关键发现：
- 深渊的真正起源
- 聆听者协会的历史真相
- 最终选择

核心对话（深渊之心）：
"你终于来了，最后的聆听者。
我等你很久了。
你想知道我是谁？
我是每一个被遗忘的声音。
我是每一句未说完的话。
我是每一个未完成的梦想。
现在，你有两个选择：
永远封印我，让我们永远沉默；
或者，与我融合，让世界听见一切。
你会怎么选？"

【分支1：封印结局】
玩家选择牺牲自己，永久封印深渊。
世界恢复和平，但再也没有声音能超越生死。

【分支2：觉醒结局】
玩家选择与深渊融合。
成为新的存在，改变世界的规则。
声音不再有生死之分，世界进入新时代。
```

### 2.6 结局设计

#### 结局A：封印

```
触发条件：战斗胜利后选择"封印"
结局动画：
- 玩家角色被光芒包围
- 深渊开始崩塌
- 声音碎片飞向天空
- 最后一刻，玩家看到所有协会成员的身影
- 他们都在微笑

结局文本：
"深渊沉默了。
不是因为它消失了，
而是因为终于有人愿意倾听它最后的独白。
艾文成为了新的回响，
但这一次，是喜悦的回响。
聆听者协会的故事结束了，
但声音永远不会停止。
只要有人在听，就有人在说。"

成就：封印者
```

#### 结局B：觉醒

```
触发条件：战斗胜利后选择"融合"
结局动画：
- 玩家与深渊融合
- 整个世界开始震动
- 颜色从暗色变为绚丽
- 声音可视化覆盖整个世界
- 玩家成为"觉醒者"

结局文本：
"深渊不再沉默。
它成为了新的生命，新的声音。
艾文不再是聆听者，
而是声音本身。
世界改变了规则：
没有死亡的声音，只有永恒的传唱。
这是祝福，还是诅咒？
时间会给出答案。
但至少……
这个世界，终于有了新的回响。"

成就：觉醒者
```

```

- 暗黑底色 + 霓虹光效

### 3.3 视觉元素规范

#### 形状语言

```
【通用规则】
- 简洁的几何形状为主
- 避免过于复杂的轮廓
- 边缘使用轻微发光效果
- 透明度用于表达虚实

【角色形状】
- 玩家：圆形头部 + 三角形身体（向前）
- 敌人：尖刺形、扭曲形
- NPC：柔和的圆形

【环境形状】
- 地板：矩形砖块或六边形
- 障碍物：三角形或菱形
- 装饰：圆形光点或波形线
```

#### 发光效果

```
【发光类型】
1. 边缘发光（Outline Glow）
   - 用于角色和敌人的轮廓
   - 颜色根据阵营变化

2. 点光源（Point Light）
   - 回响碎片
   - 技能特效
   - 环境光源

3. 体积光（Volumetric）
   - 用于重要场景
   - BOSS房间
   - 过场动画

【发光参数】
发光强度：0.3 - 1.0
发光半径：根据元素大小调整
闪烁频率：平静区域1-2Hz，战斗区域3-5Hz
```

### 3.4 视觉反馈系统

#### 战斗反馈

```
【攻击命中】
- 目标闪白 50ms
- 产生粒子爆发
- 屏幕轻微震动

【受到伤害】
- 屏幕边缘红色闪烁
- 角色闪红
- 生命值数字跳动

【暴击】
- 更大的粒子效果
- 屏幕震动加强
- 数字显示"暴击！"

【闪避】
- 残影效果
- 速度线
- 无敌帧闪烁
```

#### 环境反馈

```
【危险警告】
- 地面危险区域：红色边框 + 闪烁
- 即将到来的攻击：红色圆形预判

【可交互物品】
- 轻微上下浮动
- 周围光粒环绕
- 靠近时发光加强

【隐藏区域】
- 轻微的颜色差异
- 特殊的背景图案
```

### 3.5 素材规格

#### 角色素材

```
【玩家角色】
- 尺寸：64x64 像素
- 动画帧数：站立4帧、行走6帧、攻击4帧、受伤2帧
- 格式：PNG + Alpha通道

【敌人】
小型敌人：32x32 像素
中型敌人：64x64 像素
BOSS：128x128 或 256x256 像素
```

#### 环境素材

```
【地板】
- 单块：64x64 像素
- 重复拼接无接缝
- 每种地形至少3种变体

【墙壁/障碍】
- 高度：64像素 或 128像素
- 有碰撞和无碰撞版本

【装饰物】
- 大小可变：16x16 到 64x64
- 不影响碰撞
```

#### 特效素材

```
【粒子效果】
- 使用 Godot 粒子系统
- 备用：32x32 或 16x16 的粒子图

【UI元素】
- 图标：32x32 或 64x64
- 按钮：可变尺寸
- 数字/文字：使用字体渲染
```

### 4.2 基础操作

```
【键盘布局 - WASD】
W / ↑    - 向上移动

### 4.3 战斗系统

#### 伤害计算

```
【基础公式】
最终伤害 = (攻击方攻击力 - 防御方护甲) × 伤害类型倍率 × 暴击倍率

【具体数值】
基础攻击力：20
基础护甲：5
伤害类型倍率：普通 1.0x，弱点 1.5x，抗性 0.5x
暴击倍率：1.5x（可提升）

【护甲机制】
护甲减少受到的直接伤害
每个护甲点减少1点伤害（上限：减少50%）
```

#### 攻击类型

| 类型 | 描述 | 示例 |
|------|------|------|
| **普通攻击** | 最基础的输出手段 | 聆听者的法杖攻击 |
| **技能攻击** | 消耗技能，有特殊效果 | 音波冲击、战鼓连击 |
| **范围攻击** | 对区域内所有敌人造成伤害 | 怒吼、寂静领域 |
| **投射攻击** | 发射弹射物 | 回响蝙蝠的声波弹 |

#### 状态效果

```
【负面效果】
- 眩晕：无法移动和攻击，持续1-3秒
- 减速：移动速度降低30%，持续3秒
- 流血：每秒损失生命，持续5秒
- 燃烧：每秒损失生命，附加伤害
- 中毒：每秒损失少量生命，持续时间长

【正面效果】
- 护盾：吸收一定量伤害
- 加速：移动速度提升30%
- 攻击提升：伤害增加25%
- 生命偷取：攻击恢复生命
- 无敌：免疫所有伤害
```

   - 技能循环：按照预设顺序释放技能
   - 条件触发：满足特定条件时释放
   - 随机性：增加不可预测性

### 7.3 房间类型

```
【普通房间】
- 最基础的战斗房间
- 击败所有敌人后开门
- 随机出现1-3波敌人
- 房间清理后可能掉落回响币或碎片

【精英房间】
- 包含精英敌人
- 难度较高
- 更好的掉落
- 需要钥匙或特殊条件

【商店房间】
- 用回响币购买物品
- 商品随机刷新
- 可购买：装备、技能、药水、钥匙

【事件房间】
- 随机事件
- 可能是好事也可能是陷阱
- 需要做出选择

【篝火房间】
- 区域内的休息点
- 可以恢复生命
- 可以查看区域地图
- 可以触发对话

【隐藏房间】
- 特定条件触发
- 通常有宝箱
- 可能有危险
```

### 7.4 关卡生成

```
【房间生成算法】
1. 从起始房间开始
2. 根据连接规则生成下一个房间
3. 确保最终能到达BOSS房间
4. 填充中间房间
5. 验证路径可行性

【连接规则】
- 每个房间至少1个门
- 最多4个门
- 相邻房间类型有逻辑关系
- 存在分支路径

【难度曲线】
房间1-2：简单遭遇
房间3-4：普通遭遇
房间5-6：困难遭遇
精英房：挑战
BOSS房：考验
```

## 10. 音效与音乐

### 10.1 声音设计哲学

```
"在这个游戏里，声音不仅是听觉，更是视觉"

核心概念：
- 声音可以被"看见"
- 每个敌人有独特的声音特征
- 战斗节奏通过声音传递
- 氛围通过音乐塑造
```

### 10.2 背景音乐系统

```
【菜单音乐】
- 风格：空灵、神秘
- 乐器：合成器 + 人声吟唱
- 循环：无缝循环
- 时长：3-4分钟

【区域音乐】
每个区域有专属音乐主题：

森林：自然 + 诡异
熔岩：激烈 + 压迫
深海：低沉 + 广阔
骸骨：阴森 + 神圣
深渊：混沌 + 史诗

【战斗音乐】
- 平时：轻微的背景变化
- 精英：节奏加快
- BOSS：专属战斗音乐

【状态音乐】
低血量：心跳 + 紧迫
安全区域：平静旋律
剧情：叙事性音乐
```

### 10.3 音效设计

```
【脚步声】
玩家：轻柔的脚步（聆听者）、沉重的脚步（喧嚣者）、无声（沉默者）
敌人：根据体型变化

【攻击音效】
普通攻击：短促的打击音
技能：各有特色
暴击：更响亮 + 特殊音色
击杀：短暂的惨叫

【UI音效】
按钮悬停：轻微的高音
按钮点击：确认音
获得物品：提升音
升级：胜利音
警告：警示音
```

### 10.4 声音可视化

```
【核心视觉概念】

声波效果：
- 攻击：扇形/圆形声波扩散
- 受伤：红色波形
- 收集：光粒汇聚

敌人声音标志：
- 每个敌人有独特的声音可视化
- 攻击前有声音提示（可见的警告）

音乐可视化：
- 背景随音乐节奏轻微变化
- 关键时刻音乐触发视觉特效
```
